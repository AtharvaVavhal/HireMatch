# Contributing to HireMatch

Thank you for your interest in improving HireMatch! We welcome contributions from the community.

## How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** to your local machine.
3.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feat/your-feature-name
    ```
4.  **Install development dependencies**:
    ```bash
    pip install -r requirements-dev.txt
    ```
5.  **Make your changes** and ensure they follow our coding standards.
6.  **Write tests** for your changes.
7.  **Run the test suite** to ensure everything is working:
    ```bash
    make test
    ```
8.  **Format your code**:
    ```bash
    make fmt
    ```
9.  **Commit your changes** with a clear and descriptive message.
10. **Push your branch** to your fork:
    ```bash
    git push origin feat/your-feature-name
    ```
11. **Open a Pull Request** against the `main` branch of the original repository.

## Coding Standards

*   Use `black` for formatting.
*   Use `isort` for import sorting.
*   Write clear, documented code with docstrings.
*   Ensure all new features are covered by tests.

## Feedback

If you find a bug or have a feature request, please open an Issue on GitHub.
